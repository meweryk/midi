🎛️ ПАРАМЕТРЫ СИНТЕЗА

СЕЛЕКТОР PHI (октавный множитель)
• φ = 1.618 — золотое сечение (по умолчанию)
• 2 — классическая октава
• π = 3.1416 — число Пи
• e = 2.7183 — число Эйлера
Определяет множитель для построения частотного ряда.

ШАГ (k)
Степенной шаг для множителя PHI. Формула: freq × PHI^(i × k)
Диапазон: 1-6. Меньшие значения дают более плотный спектр.

ОБЕРТОНЫ (n)
Количество уровней разложения. Формула: freq × PHI^(i × k), где i = 0..n
Диапазон: 0-10. Больше обертонов — насыщеннее звук.

ГАРМОНИКИ
Пользовательские множители через запятую. Пример: 3/2, 6/5, 2.5, 5
При вводе гармоник поля ШАГ, ОБЕРТОНЫ и PHI блокируются.
Всегда добавляется гармоника 1 по умолчанию.

💾 ОСОБЕННОСТИ СОХРАНЕНИЯ И ГЕНЕРАЦИИ

1. Адаптивная частота дискретизации
При загрузке страницы программа определяет максимальную частоту дискретизации вашей системы и настроек браузера. В зависимости от этого динамически расширяется поле ввода частот и доступные опции сохранения WAV. Для частот выше 48 кГц на десктопе требуется установить флаг chrome://flags/#web-audio-sample-rate в значение 96 или 192 кГц.

2. Энергетическая нормализация обертонов (режим PHI)
При использовании параметров ШАГ и ОБЕРТОНЫ амплитуда каждого последующего обертона уменьшается пропорционально увеличению частоты. Это сохраняет энергетическую тождественность каждого обертонного ряда по отношению к основному ряду частот. Формула: amp = rawAmp / PHI^(i * k), где i — номер обертона.

3. Равная амплитуда гармоник (режим ГАРМОНИКИ)
При использовании пользовательских ГАРМОНИК амплитуда каждой гармоники равна амплитуде основной частоты. Это проявляет энергонасыщенность более высоких частот, делая спектр более плотным и насыщенным в верхнем регистре.

📀 СИСТЕМА РЕНДЕРИНГА WAV

Используется OfflineAudioContext — точность частот идентична реальному воспроизведению.
Параметры: 2 канала, 48 кГц (или выше при поддержке), до 180 сек.

💾 ЛОКАЛЬНОЕ ХРАНЕНИЕ

Пресеты сохраняются в localStorage браузера в формате JSON. Библиотека под окном ввода.

🎮 ИНТЕРФЕЙС

  • SAVE .WAV — генерация файла (48+ кГц / 24-32 бит)
  • SAVE PRESET — сохранение текущей конфигурации
  • PRESET LIBRARY — загрузка/удаление сохранённых пресетов

Важно: Запись происходит на максимальной скорости процессора с сохранением нулевой фазы для всех осцилляторов.
